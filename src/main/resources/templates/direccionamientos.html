Modo Real y Modo Protegido

En la arquitectura x86, se utilizan dos modos de operación para la CPU: el Modo Real y el Modo Protegido.

Modo Real

El Modo Real es el modo de operación original de los procesadores x86 y es compatible con versiones antiguas del sistema operativo DOS. En este modo, el procesador tiene acceso a toda la memoria física del sistema y puede ejecutar cualquier instrucción que sea compatible con la arquitectura x86.

Modo Protegido

El Modo Protegido es un modo de operación más avanzado que fue introducido en los procesadores 386. En este modo, el procesador protege la memoria del sistema y previene que las aplicaciones accedan a áreas de memoria que no les corresponden. Además, el Modo Protegido permite la ejecución de aplicaciones multitarea y multiproceso.

Diferencias entre Modo Real y Modo Protegido

Característica Modo Real Modo Protegido
Tamaño máximo de memoria 1 MB Depende del sistema operativo
Acceso a la memoria Acceso completo a toda la memoria física del sistema Acceso limitado a áreas específicas de memoria
Protección de memoria No hay protección de memoria Protección de memoria mediante la segmentación y paginación
Compatibilidad con versiones antiguas del sistema operativo DOS Sí No

Cambio entre Modo Real y Modo Protegido

El cambio entre Modo Real y Modo Protegido se realiza mediante una instrucción especial llamada "JUMP", que permite que el procesador salte a una nueva posición en la memoria y cambie de modo de operación.

Interrupciones

Las interrupciones son eventos que pueden ser generados por hardware o software, y que interrumpen el flujo normal de ejecución de un programa. Su uso permite una mejor gestión de recursos y una mayor flexibilidad en el diseño de sistemas.

Existen varios tipos de interrupciones, como las de hardware, que son generadas por dispositivos externos al procesador, o las de software, que son generadas por el propio programa en ejecución. Las interrupciones también pueden ser máscarales, lo que significa que pueden ser habilitadas o deshabilitadas por el sistema.

Tabla de tipos de interrupciones

Tipo Descripción
Hardware Generadas por dispositivos externos al procesador, como teclados, mouse, discos duros, entre otros.
Software Generadas por el propio programa en ejecución, como por ejemplo cuando se realiza una división por cero.
Máscarales Pueden ser habilitadas o deshabilitadas por el sistema, lo que permite un mayor control sobre su uso.

Proceso de Arranque y carga del S.O.

En esta unidad se estudia el proceso de arranque y carga del Sistema Operativo (S.O.), el cual se divide en varias etapas que se describen a continuación:

Etapa Descripción
BIOS El BIOS (Basic Input/Output System) es el primer programa que se ejecuta al encender la computadora. Se encarga de realizar las pruebas de hardware y detectar los dispositivos de almacenamiento para cargar el Sistema Operativo.
MBR El MBR (Master Boot Record) es una pequeña porción del disco duro que se utiliza para almacenar el cargador de arranque (bootloader) del S.O. Se encuentra en el primer sector del disco duro y se encarga de cargar el bootloader del S.O. que se encuentra en otra parte del disco duro.
Bootloader El bootloader es el programa que se encarga de cargar el núcleo (kernel) del S.O. en la memoria RAM. También permite al usuario seleccionar el S.O. que desea iniciar en caso de que existan varios sistemas operativos instalados en el mismo equipo.
Núcleo El núcleo del S.O. es el componente principal que se encarga de controlar los recursos del sistema (CPU, memoria, dispositivos de entrada/salida, etc.) y proporcionar una interfaz de programación para que las aplicaciones puedan interactuar con el hardware.

Mapa de la memoria

En esta unidad se explican los conceptos básicos relacionados con el mapa de memoria, que es una representación de la organización de la memoria de una computadora. A continuación, se presenta un resumen de los temas tratados en la unidad:

Concepto Descripción
Dirección de memoria Es una referencia única a una ubicación de la memoria.
Espacio de direcciones Es el conjunto de todas las posibles direcciones de memoria que pueden ser referenciadas por un programa.
Tamaño de palabra Es el número de bits que se pueden procesar simultáneamente en una operación aritmética o lógica.
Mapa de bits Es una representación gráfica del uso de la memoria, donde cada bit representa una ubicación de memoria y su valor indica si está libre o ocupada.
