Siguiente

Resumen


El concepto de virtualización hace referencia a una tecnología que permite la ejecución de varias máquinas virtuales sobre una máquina física con el objetivo de aprovechar al máximo los recursos de un sistema y que su rendimiento sea mayor. Es importante destacar que a cada una de las máquinas virtuales se le pueden asignar unos recursos (memoria, unidades de almacenamiento, procesador…) y ejecutan una copia propia de sistema operativo (Windows, Linux…).
Cuando virtualizamos, creamos un entorno informático virtual en el que cada máquina puede cumplir funciones diferentes (servidor web, servidor de archivos…) aun encontrándose todas en la misma máquina física. Esto nos ofrece la posibilidad de crear instancias con distintos sistemas operativos en un único servidor, utilizando así menos servidores para ejecutar una mayor cantidad de aplicaciones.
Una de las características principales de la virtualización es que cuando ejecutamos programas en dicho entorno, estos creen estar ejecutándose en el ámbito nativo, es decir, en un solo ordenador de uso exclusivo para ellos. Pero, en realidad, se encuentra encapsulado dentro del mismo sistema operativo donde se ejecuta la virtualización. Además de lo anterior, la virtualización hace que, aunque se ejecuten varios sistemas operativos diferentes, ninguno de ellos afecte al desempeño del otro o de la máquina principal que ha sido dividida para crear el entorno virtual.
La virtualización se basa en el uso de un software llamado hipervisor o monitor de máquina virtual, que permite la creación y gestión de máquinas virtuales. El hipervisor se instala en un servidor físico y se encarga de asignar y gestionar los recursos, como la CPU, la memoria y el almacenamiento, entre las máquinas virtuales.

Caracteristicas de la virtualización:

  • Aislamiento: Cada máquina virtual se ejecuta de forma aislada, lo que significa que un fallo o problema en una máquina virtual no afectará a las demás.

  • Consolidación: Varias máquinas virtuales pueden ejecutarse en un solo servidor físico, lo que permite una mejor utilización de los recursos y una reducción en el número de servidores físicos necesarios.S

  • Migración en vivo: Las máquinas virtuales se pueden mover de un servidor físico a otro sin interrupción del servicio, lo que permite una mayor flexibilidad y disponibilidad.

    VirtualBox: Plataforma de Virtualización

    VirtualBox es un software para virtualización, también conocido como hipervisor de tipo 2, que se utiliza para virtualizar sistemas operativos dentro de nuestro ordenador existente, creando lo que se conoce como máquina virtual. Un hipervisor de tipo 2 se diferencia de los de tipo 1 en que necesita un sistema operativo para funcionar, a diferencia de los de tipo 1 en los que el propio hipervisor funciona sobre el hardware, o máquina host.

    Existen varios factores a tener en cuenta cuando necesitamos hacer uso de un hipervisor, pero generalmente VirtualBox, y los hipervisores de tipo 2 en general, se usan cuando no necesitamos destinar la máquina host por completo para realizar tareas de virtualización. Por ejemplo, a la hora de probar software o simplemente confeccionar algunas de las guías que podemos encontrar aquí en Geeknetic, podemos aislar el sistema operativo host del sistema operativo invitado.

    VirtualBox, a diferencia de alternativas como VMWare, es gratuito, por lo que no es necesario pagar una licencia para poder tener acceso a este hipervisor. Aun así, nos permite realizar todas las tareas necesarias para mantener un entorno virtual, independientemente del uso que le queramos dar.

    Su compatibilidad se extiende a prácticamente todas las versiones de Windows, un gran número de versiones de macOS y muchas distribuciones de Linux. También podremos encontrar perfiles para sistemas menos comunes como Solaris, OS/2, DOS, QNX, Novell, BeOS y otros sistemas operativos que queramos probar o utilizar sin tener que montar un equipo específicamente para ellos.

    Si bien VirtualBox se suele ver como un producto orientado al usuario doméstico, la realidad es que es capaz de competir con alternativas de pago a la hora de hacer funcionar servidores o aplicaciones embebidas. Por lo tanto, en determinados casos, VirtualBox puede ser una muy buena opción, especialmente si no necesitamos algunas funciones exclusivas de algunos de sus competidores, entre los cuales encontramos VMWare, Microsoft Virtual PC o QEMU.

    Beneficios de VirtualBox:


    • Permite ejecutar múltiples sistemas operativos en un solo equipo físico.
    • Facilita el desarrollo y la prueba de software en diferentes entornos sin afectar el sistema principal.
    • Brinda aislamiento de recursos, lo que mejora la seguridad y la estabilidad de los sistemas virtuales.
    • Permite la migración y el clonado de máquinas virtuales para una mayor flexibilidad y escalabilidad.

    Características de VirtualBox:

    • Compatibilidad con una amplia gama de sistemas operativos anfitriones y huéspedes.
    • Funcionalidades avanzadas, como instantáneas, redimensionamiento dinámico de discos virtuales y gestión de redes.
    • Soporte para extensiones y complementos que amplían las capacidades de VirtualBox.

    Desventajas de VirtualBox:

    • El rendimiento de los sistemas virtuales puede ser inferior al de las instalaciones nativas.
    • Algunas características avanzadas pueden requerir ajustes de configuración y conocimientos técnicos adicionales.
    • La gestión de recursos puede ser compleja en entornos con múltiples máquinas virtuales.

    Guía de Instalación de VirtualBox:

    1. Descarga el instalador de VirtualBox desde el web oficial.
    2. Ejecuta el archivo de instalación y sigue las instrucciones del asistente.
    3. Una vez instalado, abre VirtualBox y crea una nueva máquina virtual.
    4. Configura los detalles de la máquina virtual, como el sistema operativo y la cantidad de memoria asignada.
    5. Finalmente, instala el sistema operativo en la máquina virtual siguiendo los pasos habituales.
